Workflow

Pick a Workflow mode at the top of the page — it sets where the COLA application fields come from:

  • Upload COLA form — upload the approved COLA application and its stated values are read into the COLA application fields automatically.
  • Extract from label — skip the COLA form and click Extract from Label to pre-fill the COLA application fields from the label image instead.
  1. Choose the product category and origin. The required fields adjust to match.
  2. Upload the label artwork — one file, or the front and back as separate files (order doesn't matter).
  3. Review and correct the COLA application fields so they match the approved application.
  4. Click Verify Against Label. Each field is marked against the label, and the government warning is checked separately.
  5. Use Batch mode when uploading multiple review items at once.

Uploads

  • Accepted files: PDF, PNG, JPEG, and WebP.
  • Maximum file size: 10 MB per file.
  • A single PDF or image can contain all label panels for one review item.
  • Use Choose File for normal file selection, or Drag & Drop when dragging files from your desktop.

Batch

  • Batch accepts multiple PDF or image files at once and processes them in parallel.
  • Each batch file is treated as one review item.
  • If a label's front and back are separate images, use Add second image on that row so both panels are verified as one item — or combine them into a single PDF or image first.
  • Set the Category and Origin per row, so one batch can mix wine, malt beverage, and distilled spirits, domestic and imported.
  • Every file is automatically extracted and verified, so each row lands on a verdict: Pass (all required fields present, the government warning compliant, and label checks passed) or Needs attention.
  • Open a row in the editor to see the field-by-field results and re-verify after correcting the COLA application fields.
  • CSV or Excel import is a future enhancement.

Notes

  • Government warning text is checked for exact wording, punctuation, and capitalization.
  • Bold type cannot be verified because extraction returns text, not typography metadata.
  • In batch mode the expected and label values both come from the label, so a row's verdict is a completeness and compliance check; upload the COLA form or edit the fields to compare against a different approved value.
